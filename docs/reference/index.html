This is a reference to the routines available in the freemco NES Corelib, as well as a reference to any tools you may want to use in developing NES games.

Tools

When developing NES games, it's useful to have tools around. I mean yeah, you can write an entire NES game with a hex editor if you wanted to, but it's really tedious and requires you to have memorized damn near everything.

Compilers

There are quite a few 6502-targeting compilers out there, but I've only tested and developed this code on ASM6 (and ASM6f).

ASM6

ASM6 is a 6502 compiler and linker by Loopy. The source code is a single C file containing portable code. If you're not afraid of the command line/terminal, ASM6 is available for all platforms. (Though you may need to compile it yourself.)

ASM6f

ASM6f is a fork of ASM6 by freem (hey, that's me!). While it's not required for using the Corelib, its ability to export FCEUX-compatible .nl files may be helpful for debugging.

Graphics

YY-CHR

YY-CHR is a pretty good program for editing the graphics of old video games (assuming the graphics aren't compressed). The .NET version works better in Windows 7, while the original C++ version works better in Windows XP and Wine. The two versions are mostly similar, though the .NET version lacks the Selection tool and the half-tone pattern tools.

NES Screen Tool

Shiru's NES Screen Tool allows you to interactively place tiles, change the palettes and attributes, and also edit the tiles if you need to. Works in Wine, if you're not on Windows. (Unrelated to nametables, it also has a Metasprite builder, which you might find useful.)

Sounds and Music

By no means is this an exhaustive list. This is blatantly missing various MML-based solutions. Do people still use NerdTracker 2?

Famitracker

Famitracker is a very popular NES tracker that supports various sound extensions and has a playback engine available. This engine is good for your games... if all you need to do is play music.

Muse

thefox has made a playback library called MUSE, which is more suited to NES games. Music data is exported from Musetracker (formerly known as "Pornotracker", if you ever see references to that).

Famitone2

Shiru recently released an update to Famitone, an engine capable of handling both sounds and music at the same time.

MetalSlime's Engine

MetalSlime's sound engine is presented as a tutorial in ten parts; the first part is available here. The sound engine I'm working on for the corelib is based off of this tutorial.

GradualGames's Engine

Derek Andrews of Gradual Games has a sound engine supporting four channels of music and two channels of sound effects.

bloopageddon

bloopageddon by ReaperSMS is probably not fit for public consumption yet, based on the todo file. If you are brave enough, you might be able to do something with it.
